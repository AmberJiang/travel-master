项目名： Travel Master 
tech keywords： muti-agent framework 


背景示例： 我需要计划一个从6月 30 日 - 7 月 16 号的家庭旅行，要在日本境内游玩4个城市。包含两位60岁以上长辈，2个12岁以下小孩，4名中青年。 需要包车，不想要跟团，但是既要舒适又要不太贵。

如果直接问LLM旅游攻略，没法回答时效性高，或者过分细致的问题。 甚至有可能出现ai幻觉，回答出一些不符合实情的攻略。

如果参考他人的旅游攻略，信息过于碎片化。不能像一个能互动的人一样，问：我附近有什么关东煮好吃？ 要回答包含关东煮的餐厅，优先回答的选项需要考虑：是否曾经放入种草清单； 是否在步行距离1km以内；是否今日不营业；是否有中英菜单；是否需要预定。

那么我的选择是，使用小红书或者淘宝在售的旅行攻略，要2026最新版的，导入我的资料库。并根据用户的情况客制化旅行攻略和回答。并且要有前后文管理。每一个旅行管理的用户特点都是不一样的。应该要创建，维护人物profile。比如年龄，精力情况，饮食偏好，交通偏好。


输入：
要能接收碎片信息并导入数据库，数据要分类放好，以后要能删除。信息通常是：小红书分享，大众点评页面，图文攻略，或者纯文字。

输出：
文字回复，地图链接 

MVP:
跑通一个从导入知识库到输出回答的流程，如果问不知道的问题，回复：我不知道，你想要我联网查询吗？
